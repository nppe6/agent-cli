# Frontend Specs

Frontend agents should start here before changing client-side code.

## Pre-Development Checklist

- Read `directory-structure.md` for where UI code should live.
- Read `type-safety.md` before changing TypeScript or schemas.
- Read `component-guidelines.md` before creating or changing components.
- Read `hook-guidelines.md` before changing reusable hooks.
- Read `state-management.md` before changing client state.
- Read `quality-guidelines.md` before final verification.

## Quality Check

- UI code follows existing component and state patterns.
- Types are explicit at boundaries.
- Loading, empty, error, and disabled states are handled where applicable.
- Tests or manual verification cover the changed interaction.
